# 429 Too Many Requests 재시도 구현 계획

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** HTTP 429 에러 발생 시 lastCheckInDate를 갱신하지 않아 다음 자동 출첵(30분 후)에서 재시도되도록 한다.

**Architecture:** HTTP 레이어에서 429를 감지하여 전용 에러를 throw하고, 비즈니스 레이어에서 catch하여 결과에 기록, 스토어에서 해당 retcode일 때 lastCheckInDate 갱신을 건너뛴다.

**Tech Stack:** TypeScript, Chrome Extension MV3, tsyringe DI, i18next

**Spec:** `docs/superpowers/specs/2026-03-27-429-retry-design.md`

### Task 5: CallCheckInApiService에서 TooManyRequestsError 처리

**Files:**
- Modify: `src/apps/background/service/check-in/CallCheckInApiService.ts:35-86`

- [ ] **Step 1: import 추가**

파일 상단에 추가:
```typescript
import { TooManyRequestsError } from '@src/shared/errors/TooManyRequestsError';
```

- [ ] **Step 2: httpWithCookie 호출을 try-catch로 감싸기**

`execute` 메서드의 for 루프 내부에서, `const url = ...` 부터 switch 문 끝까지를 try-catch로 감싼다:

```typescript
for (const checkInTarget of checkInTargetList) {
  const { actId, checkInAPIUrl, ltoken, ltuid } = checkInTarget;
  const headers = {};
  if (actId === GameActId[GameKey.ZZZ]) {
    headers['x-rpc-signgame'] = 'zzz';
  }

  try {
    const url = `${checkInAPIUrl}?lang=${getUrlLocale()}`;
    const resp = await httpWithCookie(
      url,
      {
        credentials: 'include',
        method: 'POST',
        body: JSON.stringify({ act_id: actId }),
        headers,
      },
      { ltoken, ltuid },
    );
    const { retcode, message } = resp;
    switch (retcode) {
      case ApiRetCode.Success:
        ga.fireEvent('체크인성공', { act_id: actId });
        checkInResultList.push(this.successResponse(resp, actId, ltuid));
        break;
      case ApiRetCode.AlreadyCheckIn:
        ga.fireEvent('이미체크인완료', { act_id: actId });
        checkInResultList.push(this.successResponse(resp, actId, ltuid));
        break;
      default: {
        captureApiException(new CheckInError(`${retcode}: ${message}`), url);
        const errorMsgKey = ErrorMessageKey[retcode as ApiRetCode];
        let msg = errorMsgKey ? i18n.t(errorMsgKey) : message;
        if (
          retcode === ApiRetCode.AuthExpired ||
          retcode === ApiRetCode.NotLoggedIn
        ) {
          msg = `${msg}\n${i18n.t('common.re_register_account')}`;
        }
        checkInResultList.push({
          actId,
          ltuid,
          retCode: retcode,
          lastCheckInMessage: msg,
        });
        break;
      }
    }
  } catch (e) {
    if (e instanceof TooManyRequestsError) {
      checkInResultList.push({
        actId,
        ltuid,
        retCode: ApiRetCode.TooManyRequests,
        lastCheckInMessage: i18n.t('common.too_many_requests'),
      });
    } else {
      throw e;
    }
  }
}
```

핵심: `TooManyRequestsError`만 catch하여 결과에 기록하고 루프를 계속 진행. 다른 에러는 re-throw.
